###  [Blog 1 - How Bedrock Streaming optimizes AWS costs](3.1-Blog1/)
This blog introduces how Bedrock Streaming optimizes AWS computing costs by applying a multi-axis FinOps strategy. The article covers the combination of Amazon EC2 Spot Instances, AWS Graviton Processors for Kubernetes nodes, along with fine-tuning data services like Amazon DynamoDB and Amazon S3. Readers will learn how to optimize network and storage costs to achieve maximum financial efficiency in the cloud.

###  [Blog 2 - Achieving least privilege access for Amazon Route 53 Profiles](3.2-Blog2/)
This blog demonstrates how to apply granular IAM policies for Amazon Route 53 Profiles to enforce least privilege access for multi-account DNS management. You will learn how to restrict permissions by resource type, domain name, and VPC through practical scenarios for application, security, and network teams, thereby reducing security risks and governance vulnerabilities.

###  [Blog 3 - Building highly available Oracle databases with Amazon FSx for NetApp ONTAP](3.3-Blog3/)
This blog guides how to build a highly available (HA) Oracle database architecture on AWS. The solution combines Amazon FSx for NetApp ONTAP shared storage with Auto Scaling groups, automated orchestration via AWS Lambda, and configuration management using Systems Manager Parameter Store. Readers will understand how to eliminate traditional HA complexity, automate fast failure recovery, and ensure configuration consistency for databases.

###  [Blog 4 - The big picture: Unified monitoring for AWS Parallel Computing Service](3.4-Blog4/)
This blog introduces a new observability solution for AWS Parallel Computing Service (AWS PCS) to optimize high-performance computing (HPC) environments. You will discover how to combine Amazon Managed Grafana and Amazon Managed Service for Prometheus for real-time tracking of resource allocation, job performance, and GPU/CPU consumption. As a result, users can easily resolve performance bottlenecks and optimize costs.

###  [Blog 5 - Multi-Region event-driven failover architecture with Amazon EventBridge and Route 53](3.5-Blog5/)
This blog describes how to build a highly resilient multi-region event-driven architecture using an active-passive model. By leveraging Amazon EventBridge, Amazon API Gateway, Amazon Route 53 health-based failover, and the automatic replication of Amazon DynamoDB Global Tables, the article guides you on how to ensure high system availability, minimize latency, and enable automatic failover during outages.

###  [Blog 6 - Building a scalable user search layer on Amazon Cognito](3.6-Blog6/)
This blog demonstrates how to extend Amazon Cognito with an advanced user search layer by integrating AWS Lambda, Amazon DynamoDB Streams, and Amazon OpenSearch Serverless. The solution provides flexible search capabilities with custom attributes, fuzzy matching, and complex filtering. Thus, the article guides you to build an automatically synchronized architecture to deliver high-speed query experiences for user directories.
